# 分流板大类数据字典 — 前端对接

数据来自《热流道系统》Excel/截图中**分流板大类**（属性类、桥板、分流板主体、线架等），存于 **`sel_dict_category` / `sel_dict_item`**，分类 `code` 以 **`hrspec_mfld_`** 开头，与扁平行热流道规格字段（`hrspec_*` 无 `mfld`）独立。

## 拉取全部分流板明细选项包

```http
GET /api/v1/selection-catalog/dict/manifold-detail-options
Authorization: Bearer <token>
```

权限：`selection:read`。

响应与模具/热流道规格字典包相同：`{ "categories": { "<code>": [ { "id", "label", "sort_order" } ] } }`。

前端封装：`import { fetchManifoldDetailDictBundle } from "@/lib/selectionCatalogDictApi"`。

## 分类 `code` 一览（与 `backend/app/constants/sel_manifold_detail_dict.py` 一致）

| `code` | 含义（截图列） |
|--------|----------------|
| `hrspec_mfld_process` | 工艺 |
| `hrspec_mfld_bridge_style` | 桥板-分流板搭桥样式 |
| `hrspec_mfld_bridge_channel_diameter` | 桥道直径 |
| `hrspec_mfld_manifold_thickness` | 分流板厚度 |
| `hrspec_mfld_runner_layout` | 流道走向示意图（分流板主体） |
| `hrspec_mfld_point_coding` | 点位编码 |
| `hrspec_mfld_manifold_runner_diameter` | 分流板流道直径（分流板主体，φ 记法） |
| `hrspec_mfld_runner_layers` | 流道层数 |
| `hrspec_mfld_spacer_block` | 垫块 |
| `hrspec_mfld_plate_disc_spring` | 板上碟簧 |
| `hrspec_mfld_center_locating_pin` | 中心定位销 |
| `hrspec_mfld_anti_rotation_pin` | 防转销 |
| `hrspec_mfld_plug_regular` | 堵头-常规堵头 |
| `hrspec_mfld_plug_flat` | 堵头-平面堵头 |
| `hrspec_mfld_plug_insert_basic` | 堵头-镶件堵头-基本型 |
| `hrspec_mfld_plug_insert_t` | 堵头-镶件堵头-T型 |
| `hrspec_mfld_plug_insert_i` | 堵头-镶件堵头-I型 |
| `hrspec_mfld_plug_insert_l` | 堵头-镶件堵头-L型 |
| `hrspec_mfld_water_connector` | 线架-水接头 |
| `hrspec_mfld_oil_connector` | 线架-油接头 |

**说明**

- **分流板材质**仍用既有 `GET …/dict/hot-runner-spec-options` 中的 `hrspec_manifold_material`，本包不重复。
- **法向分流板**与「分流板主体」选项相同：前端直接复用上表同一批 `code`。
- 截图中无选项的列（如部分线架项）未建分类；后续可在「选型字典」页对对应 `code` **新增字典项**，或再发迁移补种子。

## 按分类维护

与全局字典相同：`GET/POST/PATCH/DELETE …/dict/categories`、`…/dict/items?category_code=hrspec_mfld_process` 等（需 `selection:write` 写操作）。
