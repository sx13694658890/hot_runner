# 热流道技术管理端 — 业务需求规格说明书（BRD/PRD）

**文档版本**：v1.0  
**适用范围**：企业技术管理端（Web 管理后台）及相关移动端能力中与「技术、研发、设计、标准化、工艺与现场、技术 PMO」相关的功能域  
**编制说明**：本文档依据组织架构与岗位职责梳理为可落地的功能需求与数据对象边界，供产品、研发、测试与实施对齐。

---

## 1. 文档目的与范围

### 1.1 目的

- 将技术体系内各岗位的**权责、对接关系、软件落地诉求**转化为统一的需求条目。
- 为管理端的功能模块划分、权限模型、审批流与数据看板提供需求基线。

### 1.2 范围

- **在范围内**：技术总监决策与监控、研发中心、标准化与数据管理、项目设计、工艺与现场技术支持、技术项目管理组在系统中的协同与留痕。
- **不在范围内（可作为后续迭代）**：纯财务核算、人事考勤、完整 MES 车间执行层（仅描述与 ERP/MES 的接口诉求）。

### 1.3 术语

| 术语 | 说明 |
|------|------|
| 管理端 | 面向技术体系管理人员与工程师的 Web 后台（可含配套 App 数据看板） |
| 标准库/图库 | 标准零件图纸、版本、分类、权限与生命周期管理的数据集合 |
| 选型 | 基于规则/公式/知识库的标准件与方案组合选择过程 |
| 闭环 | 问题从发现、整改、验证到归档的完整状态流转 |

## 4. 功能需求（按域）

### 4.1 统一基础能力

- **组织与岗位**：部门树、岗位职级（一至四级）、人员归属；与权限角色映射。
- **权限与数据域**：功能权限 + 数据权限（按项目、客户、零件密级、模块）。
- **审批流引擎**：可配置节点、会签/或签、代理人、超时升级；**技术总监为指定流程的最终审批节点**（可配置例外）。
- **审计与留痕**：登录、导出、下载、审批、版本发布、权限变更全审计。
- **消息通知**：站内信、邮件/企业微信（可选）、节点提醒。
- **文件与版本**：大文件上传、在线预览（按格式）、版本对比（至少元数据对比）、水印与下载策略。

### 4.2 技术总监驾驶舱

- 管理端/APP：**全局项目监控**（进度、风险、里程碑偏差）。
- **研发立项审批**队列与决策意见留痕。
- **数据看板**：研发、设计、标准库、工单与质量类核心 KPI 聚合（ drill-down 到模块）。

### 4.3 研发管理（研发中心）

- 研发项目立项、阶段、任务、里程碑；与**版本迭代审批**绑定。
- 研发任务执行：分配、工时/进度、成果物上传。
- 方案/图纸版本管理；仿真数据存档；试模复盘记录关联项目/试模单。
- **研发成果入库申请**对接标准库/零件入库流程。

### 4.4 标准化与数据管理

- 标准库权限、技术标准审批流。
- 零件**入库/淘汰**审批；失效零件标记；旧版本停用与发布锁定。
- 分类与标签体系；图纸合规性校验（规则可配置）。
- **选型规则**与**选型公式**配置（与技术标准工程师职责对齐）。
- **材料参数库**、工艺标准文件、测试报告存档及与零件/工艺方案绑定。

### 4.5 BOM 与主数据运维

- 物料编码规则与维护；图纸与物料绑定。
- BOM 自动生成规则配置；与 ERP/MES **接口同步**（增量、失败重试、对账报表）。
- 选型数据统计、零件复用率分析看板；成本参数维护（权限分级）。

### 4.6 项目设计管理

- 设计任务分配、进度监控、负载视图。
- 设计方案审批；设计质量数据看板（缺陷类型、返工率等，字段可配置）。
- 客户需求与**设计变更**全流程：申请—评估—影响分析—批准—实施—关闭。
- **智能选型工具**：规则/公式/知识库驱动；**选型存根与项目绑定**。
- 标准件快速调用；非标图纸上传与版本管理；BOM 自动生成与校对任务（设计助理）。

### 4.7 工艺与现场技术支持

- 工艺方案审批；工艺文件与零件图纸绑定；工艺批注。
- 可加工性校验规则（与设计端联动提示）。
- **试模工单**：接收、执行、报告上传、多媒体存档、与项目/图纸绑定。
- **售后工单**：溯源至项目图纸/选型记录；失效案例入库；整改闭环。
- 技术问题知识库：检索、分类、关联工单/项目。

### 4.8 技术 PMO

- 项目模板（阶段、交付物、角色 RACI）。
- 全周期监控、**风险预警规则**（阈值、逾期、资源冲突等可配置）。
- 跨部门协同流程：需求、变更、问题升级通道统一留档。
- 项目资料归档与复盘记录。

## 5. 非功能需求（摘要）

| 类别 | 要求 |
|------|------|
| 性能 | 列表分页、常用看板 TTI 可接受；大图纸异步转码与预览 |
| 安全 | RBAC/ABAC、敏感字段脱敏、导出审批、操作审计 |
| 可用性 | 关键审批与工单移动端可处理（与「APP 看板」诉求一致） |
| 集成 | ERP/MES/企业微信等标准接口；幂等与补偿 |
| 合规 | 标准与知识资产分级、保密协议范围内的外发控制 |

## 6. 数据对象模型（逻辑实体，初版）

- **组织**：部门、岗位、人员、角色  
- **项目**：技术项目、阶段、任务、里程碑、风险、变更单  
- **研发**：研发项目、研发任务、版本迭代、成果入库申请  
- **设计**：设计任务、方案包、设计变更单、选型会话/选型存根  
- **标准与图库**：标准件、图纸版本、分类标签、技术标准文档、失效/淘汰记录  
- **BOM 主数据**：物料、BOM 头行、编码规则、成本参数  
- **工艺**：工艺方案、工艺文件、工艺批注、可加工性校验结果  
- **现场与售后**：试模工单、试模报告、售后工单、售后报告、媒体附件  
- **知识库**：规范条目、失效案例、解决方案、关联关系（项目/图纸/工单）  
- **审批**：流程定义、实例、节点、意见、附件  
- **集成**：同步任务、映射表、错误日志  

---

## 7. 假设与依赖

- 企业已具备或可同步：**客户/项目主数据**、**物料编码体系**、**ERP 出入库与采购相关字段**（对接范围需二次确认）。
- 图纸与仿真文件格式以企业实际工具链为准（Creo/SolidWorks/CAD 等），预览与解析策略分阶段实施。
- 「智能选型」依赖标准库与规则完备度，需标准化主管与技术标准工程师参与规则治理。

---

## 8. 验收原则（概要）

- 各二级模块至少具备：核心对象 CRUD、关键流程审批、与项目/图纸/BOM 的可追溯关联。
- 技术总监可在一处查看跨模块项目状态并完成配置的最终审批。
- 试模/售后工单具备从发起到关闭的状态机与附件留痕，并可关联到项目与图纸版本。
