## 1. 背景与目标

在技术管理端通过 **侧栏「选型向导」** 进入 **`/selection-catalog/wizard`**：左侧 **Steps** 共 **十一步** 收集信息：项目 → 产品 → 模具 → 注塑机 → **模流信息** → 热流道 → **主射咀大类** → **分流板大类** → **热咀大类 / 驱动系统** → **系统存胶模数** → **零配件**。支持 **下一步 / 上一步** 与 **点击步骤条任意步跳转**；URL 同步 `?step=1`～`11`（`replace`）。

**目标**：

- 降低单页长表单的认知负担，按业务顺序引导填写。
- 与现有「模具档案 / 选型领域表」数据域一致；向导草稿字段见 `frontend/src/lib/selectionCatalogMoldPayload.ts`（`WIZARD_MOLD_ROOT_STRING_KEYS` 等）。
- 为后端向导草稿 / 提交接口预留扩展点。

### 1.1 状态存储（技术需求）

**需求（必须满足）**：选型向导在浏览器内的 **运行态草稿**（项目 / 产品 / 模具三步聚合的 `WizardDraftV1`）**必须使用 [Zustand](https://github.com/pmndrs/zustand) 作为前端状态存储**，由全局 store 持有单一数据源；**不得**再以「仅 React Context + `useState`」作为草稿的唯一来源（避免与后续跨组件订阅、中间件扩展不一致）。

**与持久化的关系**：Zustand 负责 **内存态**；同一会话内的 **落盘** 仍通过 `wizardDraftStorage.ts` 写入 **`sessionStorage`**（`saveWizardDraftPartial` / `loadWizardDraft`），与 store 变更保持同步。后续若接入 `persist` 中间件，须与现有存储键与合并策略兼容评审。

**对外 API**：业务组件通过 `useSelectionWizard()` / `useSelectionWizardOptional()`（`SelectionWizardContext.tsx`，内部 `useSelectionWizardStore` + `useShallow`）读写；或直接 `useSelectionWizardStore` 做细粒度订阅。`SelectionWizardProvider` 仅保留应用树 **挂载边界**，不承担 `useState` 草稿。

**实现文件**：`frontend/src/stores/selectionWizardStore.ts`（`useSelectionWizardStore`）。

**非目标（仍不在本期）**：

- 与 ERP 主数据自动同步、审批流。
- 离线填写与冲突合并（多用户协作编辑）。

## 4. 十一步定义与实现摘要（当前代码）

| 步 | 名称 | 实现说明 |
|----|------|----------|
| 1 | 项目信息 | `WizardProjectInfoFields`；草稿键 `project`（Zustand `useSelectionWizardStore` / `useSelectionWizard()`） |
| 2 | 产品信息 | `WizardProductInfoFields`；`product`（同上） |
| 3 | 模具信息 | `WizardMoldInfoFields`；`mold`（`moldDraft` 语义，同上）与模具根字典等 |
| 4 | 注塑机信息 | `WizardInjectionMachineFields`；品牌/型号/设备库等 |
| 5 | 模流信息 | `WizardMoldFlowCaeFields`；`GET …/dict/wizard-cae-flow-options`，`wizard_cae_*_id` |
| 6 | 热流道信息 | `WizardHotRunnerInfoFields`；`moldDraft.root` / `rootBool`；类型为「单点针阀」见 **§4.1**，「单咀」见 **§4.2** |
| 7 | 主射咀大类 | `WizardMainNozzleCategoryFields`；`GET …/dict/main-nozzle-detail-options`，`wizard_mnz_*_id` |
| 8 | 分流板大类 | `WizardManifoldCategoryFields`；`GET …/dict/manifold-detail-options`，`wizard_mfld_*_id`、法向分流板 `wizard_mfld_normal_*_id`、主体补充文本等；热流道类型为「单咀」时见 **§4.2**（本步禁止选择） |
| 9 | 热咀大类 / 驱动系统 | `WizardHotNozzleCategoryFields`（`hrspec_hnz_*`，含 **热咀咀头** 分组）；`WizardDriveSystemCategoryFields`（`hrspec_drv_*`，含 **气缸板上开孔的驱动器** 分组）；`wizard_hnz_*_id`、`wizard_drv_*_id` |
| 10 | 系统存胶模数 | `WizardSystemGlueMoldFields`；内置三档：**小于1模**、**1~3模**、**大于3模** → `wizard_system_glue_mold_count`（`lt_1` / `1_to_3` / `gt_3`） |
| 11 | 零配件 | `WizardPartsFields`；内置 **铭牌**（本公司铭牌 / XX公司铭牌）、**压线片**（标配/加厚/不需要）→ `wizard_parts_nameplate`、`wizard_parts_wire_clip` |

**说明**：第 7～9 步字典项为 **UUID** 存 `moldDraft.root`；第 10～11 步为 **内置枚举 slug**（见 `wizardLateStepsOptions.ts`）。全量草稿键由 `emptyWizardMold()` / `WIZARD_MOLD_ROOT_STRING_KEYS` 初始化，并与 `sessionStorage` 合并（`wizardDraftStorage`）。

### 4.1 第 6 步：热流道类型「单点针阀」联动（已实现）

以 mold-options 字典 **`hot_runner_type`** 中显示名为 **「单点针阀」** 的项为准（与种子 `sel_mold_dict` 一致；**按 label 匹配，不写死 UUID**）。当选中该项时：

| 行为 | 说明 |
|------|------|
| **隐藏** | **分流板描述**（`runner_plate_style_id`，向导标签覆盖）、**点位编号规则**（`point_numbering_rule_id`）、**线架**（`wire_frame_needed`，三态）整行不展示。 |
| **进胶系统描述** | `gate_system_desc_id`（`gate_system_desc`）下拉中 **「法向」** 选项 **禁用**（不可选）；若此前已选「法向」，在切换为「单点针阀」或字典异步加载后归一化时 **自动清空**。 |
| **草稿一致性** | 上述隐藏字段在适用时清空，避免 `sessionStorage` 中残留与类型不符的 UUID。 |

实现位置：`WizardHotRunnerInfoFields.tsx`；与种子显示名一致的常量见 `selectionCatalogMoldPayload.ts`（`MOLD_HOT_RUNNER_TYPE_SINGLE_POINT_NEEDLE_VALVE_LABEL`、`MOLD_GATE_SYSTEM_DESC_NORMAL_DIRECTION_LABEL`）。

### 4.2 第 6 / 8 步：热流道类型「单咀」联动（已实现）

以 mold-options 字典 **`hot_runner_type`** 中显示名为 **「单咀」** 的项为准（**按 label 匹配**）。当选中该项时：

| 行为 | 说明 |
|------|------|
| **第 6 步：隐藏** | **分流板描述**（`runner_plate_style_id`）、**电磁阀位置**（`solenoid_valve_position_id`）、**驱动器**（`driver_type_id`）、**信号线接线方式**（`signal_wiring_method_id`）整行不展示。 |
| **第 6 步：进胶系统描述** | `gate_system_desc_id` 下拉中 **「法向」** 选项 **禁用**；若此前已选「法向」，在切换为「单咀」或字典异步加载后归一化时 **自动清空**（与 §4.1 共用「法向」label 常量）。 |
| **第 6 步：草稿一致性** | 切换为「单咀」时清空上述隐藏字段及 **所有 `wizard_mfld_*` 分流板大类草稿键**，避免与类型不符的 UUID 残留在 `sessionStorage`。 |
| **第 8 步：分流板大类** | 全部下拉与主体补充输入 **禁止选择/编辑**（`selectionForbidden`）；页面展示说明性 `Alert`，引导用户回到第 6 步修改热流道类型。 |

实现位置：第 6 步 `WizardHotRunnerInfoFields.tsx`；第 8 步 `SelectionCatalogWizardPage.tsx` 传入 `WizardManifoldCategoryFields` 的 `selectionForbidden` / `selectionForbiddenMessage`；热流道类型 label 解析复用 `useMoldHotRunnerTypeLabel.ts`（与字典 `hot_runner_type` 一致）。常量 `MOLD_HOT_RUNNER_TYPE_SINGLE_NOZZLE_LABEL` 见 `selectionCatalogMoldPayload.ts`。

## 5. 交互与校验策略（当前实现）

- **步骤条**：可自由跳转，**不做**跨步必填拦截。
- **下一步 / 上一步**：不在切换前做强校验（与 §5.1「严格」策略未启用一致）。
- **完成**：第 11 步点击完成仅前端 `message.success`；全量校验与落库待业务与后端对接。

---

## 6. 数据持久化与提交（分阶段）

### 6.1 当前（已实现）

- **与 §1.1 对齐**：运行态草稿由 **Zustand** `frontend/src/stores/selectionWizardStore.ts`（`useSelectionWizardStore`）持有；各步经 `useSelectionWizard()`（`frontend/src/contexts/SelectionWizardContext.tsx`，`useShallow`）或直连 store 读写；每次变更仍调用 `saveWizardDraftPartial`，与 **`sessionStorage`**（`wizardDraftStorage.ts` / `loadWizardDraft`）同步。
- `SelectionWizardProvider`（`App.tsx` 根路由包裹）：仅挂载边界，**不**以 `useState` 存草稿。
- **完成**：未调用后端；未跳转模具新建页（可在后续任务中对接）。

### 6.2 后续

- 后端 **向导草稿** `POST/PATCH`、**提交** 与 `sel_mold_info` 等表字段映射（见领域表 API 文档）。
- 第 10～11 步若需与字典/BOM 统一，可将内置选项迁入 `sel_dict_*` 并改下拉数据源。
