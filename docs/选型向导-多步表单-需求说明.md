# 选型向导（多步表单）— 需求说明

**文档类型**：产品 / 前端需求（实现前评审用）  
**版本**：v0.3  
**状态**：草案  
**关联**：`docs/模具选型-领域表-前端对接API文档.md`、`frontend/src/pages/SelectionCatalogMoldFormPage.tsx`（现有单页模具表单）

---

## 1. 背景与目标

在技术管理端增加 **侧栏入口**，进入 **选型向导** 页面：通过 **顶部进度条（Steps）** 分 **十一步** 收集信息：项目 → 产品 → 模具 → 注塑机 → **模流信息** → 热流道 → **主射咀大类** → **分流板大类** → **热咀大类** → **系统存胶模数** → **零配件**。每步填写后可通过 **下一步** 推进；同时支持 **在进度条上直接点击任意步骤** 跳转，便于核对或补录。

**目标**：

- 降低单页长表单的认知负担，按业务顺序引导填写。
- 保持与现有「模具档案 / 选型领域表」数据域一致，避免重复造字段语义。
- 为后续与后端草稿保存、提交接口对接预留扩展点。

**非目标（本期可不实现，文档仅预留）**：

- 与 ERP 主数据自动同步、审批流。
- 离线填写与冲突合并（可多用户协作编辑）。

## 2. 范围说明

| 项 | 说明 |
|----|------|
| **入口** | 侧栏新增菜单项，建议文案：**「选型向导」** 或 **「分步选型」** |
| **路由** | 建议：`/selection-catalog/wizard`（与现有 `/selection-catalog/*` 同域，便于权限 `selection:read` / `selection:write`） |
| **权限** | 与模具表单一致：进入页面至少 `selection:read`；保存/提交需 `selection:write`（若本期仅 UI 壳，可全部只读占位） |
| **步骤数** | **十一步**（见第 4 节）；进度条展示 11 个节点 |

## 3. 信息架构与导航

### 3.1 侧栏

- 在 `Layout` 的 `navItems` 中增加一项，`to: "/selection-catalog/wizard"`，`label` 见上，`perm: "selection:read"`。
- 菜单选中规则：长路径优先，确保 `/selection-catalog/wizard` 不被 `/selection-catalog` 错误吞并（与现有 `resolveMenuSelectedKey` 逻辑一致）。

### 3.2 页面布局（建议）

- **顶部**：`Steps`（Ant Design）或等价组件，十一步标题固定、顺序固定。
- **主区**：当前步骤对应的表单区域（卡片 / 分区）。
- **底部操作栏**：
  - **上一步**（第 2 步及以后显示）
  - **下一步**（第 1～10 步显示；第 11 步为 **完成** / **提交**，见第 6 节）
- **跳转**：点击 `Steps` 的某一步 **直接切换** 到该步（不要求必须先点「下一步」）。

### 3.3 URL 与步骤同步（建议）

- 使用查询参数保持可分享与刷新不丢步，例如：`/selection-catalog/wizard?step=3`（`step` 取值 `1`～`11`）。
- 切换步骤时 `replace` 更新 URL，避免历史栈过长。
- 非法 `step` 回退到 `1` 或最近一次合法步。

## 4. 十一步定义与内容边界

| 步骤 | 名称 | 说明（需求层） |
|------|------|----------------|
| 1 | 项目信息 | 与项目/客户/订单相关的上下文：如项目引用、客户、交期期望等（字段以现有 `projects` 或业务约定为准，可与「项目」模块联动） |
| 2 | 产品信息 | 产品名称、材料、克重、型腔数等；对齐现有模具表单中的 **产品维度**（参考 `PRODUCT_FIELD_META`、`SelectionCatalogMoldFormPage` 中产品区） |
| 3 | 模具信息 | 模具编号、型腔、模架、尺寸、温控等 **模具根信息**（参考 `MOLD_ROOT_*`、字典项绑定） |
| 4 | 注塑机信息 | 机台吨位、螺杆、射出能力等与注塑机相关的约束或选型参数（若领域表暂无独立聚合，本期可为结构化占位 + 备注） |
| 5 | 模流信息 | **模流分析 / CAE** 相关输入或结论：如材料牌号与模流一致说明、浇口位置结论、收缩与变形关注点、是否已出模流报告及报告编号/附件等（字段与是否对接外部 PLM/仿真系统待业务定；本期可为结构化占位 + 附件上传预留） |
| 6 | 热流道信息 | 与热流道系统相关的选项：可与 **热流道列表 / 扁平行规格** 或 `hot-runner-spec-options`、各 `*-detail-options` 字典包对齐（分阶段接入） |
| 7 | 主射咀大类 | **主射咀** 在选型结构中的大类选择或归类（与主射咀规格/字典树对齐；字段与数据源待业务与领域表评审，本期可为单选/级联占位） |
| 8 | 分流板大类 | **分流板** 大类选择或归类（与分流板规格/字典对齐；分阶段接入） |
| 9 | 热咀大类 | **热咀** 大类选择或归类（与热咀规格/字典对齐；分阶段接入） |
| 10 | 系统存胶模数 | 热流道系统 **存胶模数** 相关参数或档位（与工艺/系统能力约定对齐；本期可为数值或枚举占位） |
| 11 | 零配件 | **零配件** 清单或选型项（如密封件、加热件等；可与后续 BOM/选配表对接；本期可为多选占位 + 备注） |

**说明**：字段级清单需在评审后拆到 **接口/字段映射表**；本文档只锁定 **步骤划分** 与 **交互**。第 7～11 步为热流道信息之后的 **深化拆分**，便于与扁平行规格、字典包分步加载与校验对齐。

## 5. 交互与校验策略

### 5.1 下一步

- 点击 **下一步** 时，对 **当前步** 做前端校验（必填、格式）；失败则 **不切换步骤**，并在表单项旁或顶部 `Alert` 提示。
- 可选策略（评审二选一）：
  - **严格**：未完成当前步不允许进入下一步（点击步骤条跳转时同样校验目标步之前所有步）。
  - **宽松**：允许跳步，仅在 **提交/完成** 时统一校验；步骤条任意跳转不拦截（需在 UI 上标明「未填」状态）。

**建议默认**：步骤条 **可自由跳转**；**下一步** 仅校验当前步。**完成** 时做全量校验（与业务确认）。

### 5.2 上一步

- 不校验，直接回到上一步；已填数据保留在客户端状态中。

### 5.3 步骤条点击

- 与「下一步」行为一致：若采用严格策略，则阻止跳转并提示；若宽松策略，则直接切换。

## 6. 数据持久化与提交（分阶段）

### 6.1 MVP（建议）

- 前端用 **单页 React state**（或 `useReducer`）聚合十一步表单。
- 可选：`sessionStorage` 键如 `sel-wizard-draft`，刷新页面可恢复（注意敏感信息策略）。
- **完成** 按钮：本期可 **仅汇总展示 JSON / 跳转至现有「新建模具」** `SelectionCatalogMoldFormPage` 并预填能映射的字段（需单独开发任务）。

### 6.2 后续

- 后端增加 **向导草稿** `POST/PATCH` 与 **提交** 接口，与 `sel_mold_info` 等表写入对齐（见 `docs/模具选型-领域表-前端对接API文档.md`）。

---

## 7. 非功能需求

- **一致性**：视觉与 `SelectionCatalog*` 页面统一（圆角卡片、主色、表格/表单密度）。
- **可访问性**：步骤有明确 `aria` 标签（若用 Ant Design Steps，尽量保留语义）。
- **性能**：单步懒加载重字典包（若某步需拉 `GET …/dict/*`），避免首屏多次请求并行打满。

---

## 8. 验收标准（MVP）

1. 侧栏出现新菜单，点击进入 `/selection-catalog/wizard`（或带 `?step=`）。
2. 可见 **十一步进度条**，标题与本文第 4 节一致。
3. **下一步 / 上一步** 可切换步骤；数据不无故丢失（同会话内）。
4. 点击 **步骤条任意一步** 可切换到对应步（与第 5 节选定策略一致）。
5. 无 `selection:read` 的用户不可见菜单或进入后提示无权限（与现有策略一致）。
